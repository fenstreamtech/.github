# Contributing

Thank you for contributing to repositories under Fenstream Technology Group governance.

## Before Contributing

Review:

- README (repo-specific)
- [SECURITY.md](SECURITY.md)
- [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
- [SUPPORT.md](SUPPORT.md)

Security vulnerabilities must be reported privately per `SECURITY.md`.

## Contribution Types

- bug fixes
- documentation improvements
- tests and reliability changes
- feature enhancements

## Issue Guidelines

Include:

- clear summary
- expected vs actual behavior
- reproduction steps
- environment details
- relevant logs or screenshots

## Pull Request Guidelines

- keep PRs focused and atomic
- include/update tests when behavior changes
- update docs for user-facing changes
- use clear, descriptive commit messages
- pass CI and required checks

## Review and Merge

Maintainers review for correctness, security, compatibility, and maintainability.  
Merge requires maintainer approval and successful required checks.

## Licensing

By contributing, you agree contributions are provided under the repository license.
