# Security Notice

## What This Covers

This notice applies to the **[Project Name]** repository and related release artifacts maintained by **[Company Name]**. []
Our security and engineering teams review and triage all reported vulnerabilities.

## Reporting a Vulnerability

We support responsible vulnerability disclosure. If you believe you have found a security issue, please report it privately.

- Preferred: GitHub “Report a vulnerability” (Security Advisory for this repository)
- Email: [Company Security Email]
- Optional security profile: `https://yourcompany.com/.well-known/security.txt`

Please include:

- the repository, file, or feature affected
- a concise vulnerability description
- reproduction steps
- expected impact and non-destructive proof-of-concept

If available, also include:

- vulnerability class (for example, OWASP category)
- logs, screenshots, or traces supporting the report
- suggested mitigation

## Response Expectations

For valid reports, our target timelines are:

- acknowledgement within 2 business days
- initial triage within 5 business days
- remediation plan or status update within 10 business days

We will provide periodic updates until closure.

## Researcher Guidelines

When testing and reporting, you must not:

- break the law
- access, exfiltrate, or retain unnecessary data
- modify data or persistence settings
- run destructive or high-intensity scanning
- attempt denial-of-service or service disruption
- socially engineer, phish, or physically target staff/infrastructure
- publicly disclose details before coordinated disclosure
- request payment in exchange for disclosure

## Scope Notes

In scope:

- code and configurations in this repository
- official release artifacts published by maintainers

Out of scope:

- third-party services outside our control
- forks and unofficial mirrors
- non-exploitable informational findings without security impact (best-effort review)

## Bug Bounty

[Company Name] does not currently operate a paid bug bounty program for this repository.  
We may acknowledge responsible disclosures, subject to internal policy.

## Safe Harbor

If you act in good faith, follow this notice, and avoid harm, **[Company Name]** will treat your research as authorized under this policy.
