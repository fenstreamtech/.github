# Support

## Where to Get Help

- usage questions: GitHub Discussions (or repository issues if discussions are disabled)
- defects/bugs: GitHub Issues
- security concerns: private reporting only via [SECURITY.md](SECURITY.md)

## What to Include

- repository name and version/commit
- environment details
- reproduction steps
- relevant logs/errors
- expected and actual outcomes

## Response Model

Support is provided on a best-effort basis during business days.  
Priority is given to security, reliability, and active-release issues.
